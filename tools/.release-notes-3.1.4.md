## LockpickSettings 3.1.4

In-game settings menu support, plus extra crash hardening.

**SharedModMenu support.** If you also run the optional SharedModMenu, LockpickSettings now shows up as a tab there, so you can change its settings live from one in-game menu instead of editing `config.lua`. The tab groups the live-tunable settings: the next-move hint and connection display, the full-auto toggle and the auto-solve speed, and the per-tier durability bonuses (untrained / trained / master). Edits apply immediately. This is built on the shared kit's cross-mod menu bridge and is a no-op if SharedModMenu is not installed, the mod works exactly as before on its own.

- **More native-access-violation guards.** Building on the validity re-check that 3.1.3 added for stale material handles, this build extends the same per-call `IsValid()` gating to more of the cached engine handles the adapter touches. The game can tear an object or material instance down (scene teardown / GC) between the moment the mod captures it and a later use, and touching the dangling handle raises a native crash that `pcall` cannot catch. Re-checking validity at the point of use is the only real guard, so it is now applied more widely.
- **Internal: tests run on the real runtime.** The Lua test suite now runs under PUC Lua 5.4, which is what UE4SS actually runs, instead of LuaJIT. No shipped behaviour changes from this; it just means the tests exercise the same Lua the game does.

No gameplay changes. Auto-solve is still off by default and bound to F6 (solve current lock) / Shift+F6 (full-auto every lock); `autoSolveSpeed` in `config.lua` is unchanged (default 1000 = near-instant; lower it for a human-watchable solve).

A note on the occasional F6 / F7 crash: that is a known bug in UE4SS itself (its deferred-action queue, RE-UE4SS issue #1180, fixed upstream in PR #1201), not in this mod, and 3.1.4 does not work around it. It will go away with a UE4SS build that carries the upstream fix.

### Requirements

- Gothic 1 Remake with an **experimental UE4SS build** (the bundled `dwmapi.dll` proxy loader, placed next to `G1R-Win64-Shipping.exe`). The included readme covers this.

### Install (manual)

1. Get UE4SS working (the `complete` build on the mod host bundles it; this GitHub zip is the mod only).
2. Copy the `LockpickSettings` folder into `...\G1R\Binaries\Win64\ue4ss\Mods\`. The included `enabled.txt` activates it; no `mods.txt` editing needed.
3. Start the game and check `...\ue4ss\UE4SS.log` for the `[LockpickSettings] Loaded 3.1.4` banner.

See the bundled **`readme.txt`** for all config options and uninstall.
