Crash hardening, plus an old-build compatibility alias.

**The headline fix is a crash.** A lock-piece's material handle (the one the mod tints) can be torn down by the game between when it is captured and a later re-tint (lock scene teardown / GC). Touching that dangling handle raised a native crash. Reported by a user who toggled the next-move hint (F7) on a cellar door just as the pick was ending. The fix re-checks the handle's validity before every colour read or write, covering all callers (per-tick re-tint, the F7 toggle, and auto-solve).

**Compatibility alias for the older game build.** The 2026-06-12 game update renamed the orc dig-site chest (`AM_Chest_05` -> `FM_Chest_Digginggallery_02`). The mod now keys both names to the same lock graph, so the next-move hint and auto-solve work on that chest whether you are on the new build or still on the older one.

No gameplay changes. Auto-solve is still off by default: F6 solves the current lock, Shift+F6 toggles full-auto on every lock, and `autoSolveSpeed` in `config.lua` controls the pace.

### Requirements

- Gothic 1 Remake with an **experimental UE4SS build** (the bundled `dwmapi.dll` proxy loader, placed next to `G1R-Win64-Shipping.exe`). The bundled `readme.txt` covers this.

### Install (manual)

1. Get UE4SS working (the `complete` build on the mod host bundles it; this GitHub zip is the mod only).
2. Copy the `LockpickSettings` folder into `...\G1R\Binaries\Win64\ue4ss\Mods\`. The included `enabled.txt` activates it; no `mods.txt` editing needed.
3. Start the game and check `...\ue4ss\UE4SS.log` for the `[LockpickSettings] Loaded 3.1.3` banner.

See the bundled **`readme.txt`** for all config options and uninstall.
